# シンペイ コマ・アニメーション仕様

## 1. 目的

このドキュメントは、シンペイのWeb実装におけるコマの見た目と、操作時のアニメーションを定義するための仕様である。

対象とする要素は以下。

- コマの基本的な見た目
- マスを選択したときの表現
- コマを選択したときの表現
- 通常移動時のアニメーション
- 相手のコマを挟んだときの演出
- 挟んだ相手コマを飛ばすときのアニメーション
- 不正操作時のフィードバック

## 2. コマの基本デザイン

シンペイのコマは、盤面の穴に差し込んで使う丸いピン状のコマである。

各プレイヤーは同じ形のコマを4個ずつ持ち、色によって所有者を区別する。  
コマに種類やランクの違いはなく、見た目の違いは基本的に色のみである。

## 3. コマの見た目の特徴

| 項目 | 内容 |
|---|---|
| 形 | 丸いピン状・キノコ状・画鋲のような形 |
| 上から見た形 | 円形 |
| 側面 | 短い柱のような形 |
| 上部 | つまみやすい丸い頭 |
| 下部 | 盤面の穴に差し込む短い軸 |
| 色 | プレイヤーごとに色が違う |
| 数 | 各プレイヤー4個ずつ |
| 質感 | プラスチック製のおもちゃのような質感 |
| 役割差 | 見た目上の種類差はなく、色だけで区別する |

## 4. Web UI上での表現

Web UIでは、実物のコマを完全再現する必要はない。

画面上では、コマは円形の立体的なチップとして表現する。

- プレイヤー1のコマは赤系
- プレイヤー2のコマは青系
- 上から見た円形を基本にする
- 影やハイライトを付けて、少し立体感を出す
- 盤面の穴に置かれているように見せる
- 選択可能なコマは、ユーザーがクリックできることが分かるようにする

## 5. 色の考え方

具体的な色は実装側で調整してよいが、以下の方針とする。

| 要素 | 表現 |
|---|---|
| プレイヤー1 | 赤系 |
| プレイヤー2 | 青系 |
| 選択中のコマ | 明るい縁取り、または発光 |
| 移動可能なマス | 薄いハイライト |
| 移動先として選択中のマス | 強めのハイライト |
| 不正なマス | 一瞬だけ赤系のエラー表示 |

## 6. 基本インタラクション

シンペイでは、操作対象によってUI状態を変える。

主な状態は以下。

| 状態 | 内容 |
|---|---|
| idle | 何も選択していない状態 |
| pieceSelected | 自分のコマを選択している状態 |
| cellSelected | 配置先・移動先のマスを選択している状態 |
| moving | コマが移動中の状態 |
| captured | 相手のコマを挟んだ状態 |
| relocating | 挟まれたコマを飛ばしている状態 |
| invalid | 不正な操作をした状態 |

## 7. マスを選択したときの表現

マスを選択したときは、選択されたマスが明確に分かるようにする。

### 表現方針

- 選択されたマスの周囲を光らせる
- マスの中心に薄い円形のハイライトを表示する
- 選択中であることが分かるように、軽い拡大アニメーションを入れる
- ハイライトは盤面やコマより目立ちすぎないようにする

### アニメーション

| 項目 | 内容 |
|---|---|
| 開始タイミング | マスをクリック、またはタップした瞬間 |
| 時間 | 100〜150ms |
| 動き | 少し拡大して元に戻る |
| 表現 | 薄い発光、円形ハイライト |
| 目的 | 選択された場所をユーザーに伝える |

### イメージ

```text
通常:
  ○

選択時:
  ◎
```

### CSSイメージ

```css
.cell.selected {
  transform: scale(1.08);
  box-shadow: 0 0 10px rgba(255, 255, 255, 0.6);
  transition: transform 120ms ease-out, box-shadow 120ms ease-out;
}
```

## 8. 移動可能なマスの表示

コマを選択したときは、そのコマが移動できるマスを表示する。

### 表現方針

- 移動可能なマスに薄い円形マーカーを表示する
- マーカーはコマより小さくする
- 相手のコマがある場所や移動できない場所には表示しない
- 配置フェーズでは、配置可能な空きマスを表示してもよい

### アニメーション

| 項目 | 内容 |
|---|---|
| 開始タイミング | 自分のコマを選択した直後 |
| 時間 | 120〜180ms |
| 動き | フェードイン + 少し拡大 |
| 表現 | 半透明の円形マーカー |
| 目的 | 次に選べる場所を分かりやすくする |

### イメージ

```text
移動可能:
  ・

移動不可:
  表示なし
```

## 9. コマを選択したときの表現

自分のコマを選択したときは、そのコマが現在の操作対象であることを明確にする。

### 表現方針

- コマを少しだけ浮かせる
- コマを少し拡大する
- コマの周囲にリング状のハイライトを表示する
- 操作可能なコマであることを示すため、カーソルを pointer にする

### アニメーション

| 項目 | 内容 |
|---|---|
| 開始タイミング | 自分のコマをクリック、またはタップした瞬間 |
| 時間 | 120〜180ms |
| 動き | 少し上に浮く + 少し拡大 |
| 表現 | 影を強くする、縁取りを表示する |
| 目的 | 選択中のコマを分かりやすくする |

### CSSイメージ

```css
.piece.selected {
  transform: translateY(-6px) scale(1.08);
  box-shadow: 0 8px 16px rgba(0, 0, 0, 0.25);
  transition: transform 150ms ease-out, box-shadow 150ms ease-out;
}
```

## 10. 通常移動時のアニメーション

コマを移動するときは、現在位置から移動先まで滑らかに移動させる。

### 表現方針

- コマが盤面上を滑るように移動する
- 移動中は少しだけ浮いて見えるようにする
- 移動完了時に軽く着地するような動きを入れる
- 移動が完了するまでは次の操作を受け付けない

### アニメーション

| 項目 | 内容 |
|---|---|
| 開始タイミング | 合法な移動先を選択した直後 |
| 時間 | 180〜250ms |
| 動き | 現在位置から移動先へ移動 |
| イージング | ease-in-out |
| 完了時 | 小さく沈んで戻る |
| 目的 | コマがどこからどこへ移動したかを伝える |

### イメージ

```text
移動前:
  A: ●
  B: ○

移動中:
  A: ○  →  ●  →  B

移動後:
  A: ○
  B: ●
```

### CSSイメージ

```css
.piece.moving {
  transition: transform 220ms ease-in-out;
  z-index: 10;
}

.piece.land {
  animation: piece-land 120ms ease-out;
}

@keyframes piece-land {
  0% {
    transform: scale(1.08);
  }
  100% {
    transform: scale(1);
  }
}
```

## 11. 配置時のアニメーション

配置フェーズで手元のコマを盤面に置くときは、盤面に差し込まれるような動きにする。

### 表現方針

- コマが少し上から落ちてくる
- 着地時に軽く跳ねる
- 盤面の穴に差し込まれたように見せる

### アニメーション

| 項目 | 内容 |
|---|---|
| 開始タイミング | 配置先を選択した直後 |
| 時間 | 180〜250ms |
| 動き | 上から下へ移動 + 軽いバウンド |
| 目的 | 新しく置かれたことを分かりやすくする |

### CSSイメージ

```css
.piece.placed {
  animation: piece-place 220ms ease-out;
}

@keyframes piece-place {
  0% {
    opacity: 0;
    transform: translateY(-16px) scale(0.95);
  }
  70% {
    opacity: 1;
    transform: translateY(2px) scale(1.03);
  }
  100% {
    transform: translateY(0) scale(1);
  }
}
```

## 12. 相手のコマを挟んだときの演出

相手のコマを挟んだ場合、挟まれたコマがどれか分かるように演出する。

### 表現方針

- 挟まれた相手コマを一瞬光らせる
- 挟んでいる自分の2つのコマも軽く光らせる
- 挟まれたコマを少し震わせる
- その後、飛ばす操作に移る

### アニメーション

| 項目 | 内容 |
|---|---|
| 開始タイミング | 着手後、挟み判定が成立した瞬間 |
| 時間 | 250〜400ms |
| 動き | 挟まれたコマが小さく震える |
| 表現 | 発光、点滅、軽い振動 |
| 目的 | どのコマが挟まれたかを明確にする |

### イメージ

```text
挟み成立:

自分   相手   自分
 ●  -  ○  -  ●

挟まれた相手コマ:
 ○ が震える / 光る
```

### CSSイメージ

```css
.piece.captured {
  animation: piece-captured 360ms ease-in-out;
}

@keyframes piece-captured {
  0% {
    transform: translateX(0);
    filter: brightness(1);
  }
  20% {
    transform: translateX(-3px);
  }
  40% {
    transform: translateX(3px);
    filter: brightness(1.3);
  }
  60% {
    transform: translateX(-2px);
  }
  80% {
    transform: translateX(2px);
  }
  100% {
    transform: translateX(0);
    filter: brightness(1);
  }
}
```

## 13. 挟んだコマを飛ばす操作

相手のコマを挟んだ場合、挟んだプレイヤーはその相手コマを任意の空きマスへ移動させる。

このとき、通常移動とは異なり、隣接制約や別世界への移動制約は受けない。

### UI状態

挟み成立後は、通常の手番操作ではなく、挟まれたコマの移動先を選ぶ状態に入る。

```text
state: relocating
```

この状態では、以下のようにする。

- 挟まれた相手コマを選択中として表示する
- 移動先にできる空きマスをすべてハイライトする
- 元の位置は選択不可にする
- 他のコマは操作不可にする
- 移動先を選ぶまで手番は完了しない

## 14. 飛ばすときのアニメーション

挟まれたコマを飛ばすときは、通常移動よりも大きな動きを付ける。

### 表現方針

- コマが一度上に浮く
- 放物線を描くように移動先へ飛ぶ
- 移動先で軽く着地する
- 通常移動より少し派手にする
- ただし、ゲームテンポを損なわない長さにする

### アニメーション

| 項目 | 内容 |
|---|---|
| 開始タイミング | 飛ばす先の空きマスを選択した直後 |
| 時間 | 350〜500ms |
| 動き | 上に浮く + 放物線移動 + 着地 |
| イージング | ease-in-out |
| 完了時 | 軽いバウンド |
| 目的 | 通常移動ではなく、挟まれて飛ばされたことを表現する |

### イメージ

```text
飛ばす前:
  A: ○

飛ばし中:
  A: ○  ↗  ↘

飛ばした後:
  B: ○
```

### CSSイメージ

実装上は、JavaScriptで開始位置と終了位置を計算し、`transform` で移動させる。

```css
.piece.relocating {
  z-index: 20;
  transition: transform 420ms ease-in-out;
}

.piece.relocate-land {
  animation: relocate-land 140ms ease-out;
}

@keyframes relocate-land {
  0% {
    transform: scale(1.12);
  }
  70% {
    transform: scale(0.96);
  }
  100% {
    transform: scale(1);
  }
}
```

放物線を表現したい場合は、X方向とY方向を分けてアニメーションする。

```text
outer element:
  X方向へ移動

inner element:
  Y方向に一度浮いてから落ちる
```

## 15. 複数のコマを同時に挟んだ場合

1手で複数の相手コマを挟んだ場合、挟まれたコマを順番に飛ばす。

### 表現方針

- 挟まれたコマをすべて一度ハイライトする
- その後、1個ずつ飛ばす先を選ばせる
- 現在飛ばす対象のコマだけを強くハイライトする
- 飛ばし終わったコマは通常表示に戻す
- すべてのコマを飛ばし終わったら手番を終了する

### UI状態例

```ts
type RelocatingState = {
  type: "relocating";
  player: Player;
  targets: PieceId[];
  currentTarget: PieceId;
};
```

## 16. 不正操作時のアニメーション

移動できないマスや選択できないコマをクリックした場合、エラーであることを軽く伝える。

### 表現方針

- 大きな警告は出さない
- 対象のマスまたはコマを一瞬だけ震わせる
- 移動できない理由が分かりづらい場合は、短いメッセージを出してもよい

### アニメーション

| 項目 | 内容 |
|---|---|
| 開始タイミング | 不正な操作をした瞬間 |
| 時間 | 150〜220ms |
| 動き | 小さく左右に震える |
| 表現 | 赤系の一瞬のハイライト |
| 目的 | その操作はできないことを伝える |

### CSSイメージ

```css
.invalid {
  animation: invalid-shake 180ms ease-in-out;
}

@keyframes invalid-shake {
  0% {
    transform: translateX(0);
  }
  25% {
    transform: translateX(-4px);
  }
  50% {
    transform: translateX(4px);
  }
  75% {
    transform: translateX(-2px);
  }
  100% {
    transform: translateX(0);
  }
}
```

## 17. アニメーション中の操作制御

アニメーション中は、状態の不整合を防ぐため、原則として次の操作を受け付けない。

| 状態 | 操作受付 |
|---|---|
| moving | 受け付けない |
| placing | 受け付けない |
| captured | 受け付けない |
| relocating | 移動先選択のみ受け付ける |
| invalid | 次の操作を受け付けてもよい |

特に、以下の処理中はUIをロックする。

- コマの配置アニメーション
- 通常移動アニメーション
- 挟み成立演出
- 飛ばすアニメーション
- 勝利演出

## 18. 勝利時の演出

自分のコマが3個一直線に並んだ場合、勝利ラインを分かりやすく表示する。

### 表現方針

- 勝利ライン上の3個のコマを光らせる
- 勝利ラインを線で結ぶ
- 盤面全体は少し暗くして、勝利ラインを目立たせる
- 勝者を表示する

### アニメーション

| 項目 | 内容 |
|---|---|
| 開始タイミング | 勝利判定が成立した直後 |
| 時間 | 500〜800ms |
| 動き | 勝利コマが順番に光る |
| 表現 | 発光、ライン表示 |
| 目的 | どの並びで勝ったかを伝える |

## 19. アニメーション速度の目安

全体として、テンポを悪くしないため、1つのアニメーションは短めにする。

| アニメーション | 時間の目安 |
|---|---|
| マス選択 | 100〜150ms |
| コマ選択 | 120〜180ms |
| 配置 | 180〜250ms |
| 通常移動 | 180〜250ms |
| 挟み成立 | 250〜400ms |
| 飛ばす | 350〜500ms |
| 不正操作 | 150〜220ms |
| 勝利演出 | 500〜800ms |

## 20. 実装上の注意

アニメーションは見た目の演出であり、ゲーム状態の正しさは必ずロジック側で管理する。

### 方針

- ゲーム状態は先に確定する
- UIアニメーションは確定済みの状態を表現する
- アニメーション中は必要に応じて入力をロックする
- アニメーション完了後に次の状態へ進める
- P2P対戦では、相手側にも同じイベントを送って同じ演出を再生する

## 21. P2P対戦時のイベント例

P2P対戦では、操作結果をイベントとして相手に送る。

### 配置イベント

```json
{
  "type": "place",
  "turn": 1,
  "player": "red",
  "pieceId": "R1",
  "to": "U11"
}
```

### 移動イベント

```json
{
  "type": "move",
  "turn": 9,
  "player": "red",
  "pieceId": "R2",
  "from": "U11",
  "to": "L00"
}
```

### 挟んだコマを飛ばすイベント

```json
{
  "type": "relocate",
  "turn": 9,
  "player": "red",
  "targetPieceId": "B1",
  "from": "U12",
  "to": "L22"
}
```

### 勝利イベント

```json
{
  "type": "win",
  "turn": 12,
  "player": "blue",
  "line": ["L00", "L11", "L22"]
}
```

## 22. AIに伝える用の短い要約

```text
シンペイのコマは、丸いピン状・キノコ状・画鋲状の形をした立体的なコマである。
上から見ると円形で、盤面の穴に差し込まれているように見える。
プレイヤーごとに色が異なり、コマごとの形状差や能力差はない。

マスを選択したときは、選択マスを薄く光らせ、軽く拡大する。
コマを選択したときは、コマを少し浮かせ、周囲にリング状のハイライトを出す。
移動可能なマスには、半透明の円形マーカーを表示する。

通常移動では、コマが盤面上を滑るように移動し、移動先で軽く着地する。
配置時は、コマが上から落ちて盤面の穴に差し込まれるように見せる。
相手のコマを挟んだときは、挟まれたコマを一瞬光らせて震わせる。
挟まれたコマを飛ばすときは、通常移動より大きく、少し浮いて放物線を描くように移動させる。
不正な操作をした場合は、対象を小さく震わせて操作できないことを伝える。
```
