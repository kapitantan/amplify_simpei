# シンペイ 拡張ルール案 v0.1

## 1. 追加ルールの目的

既存のシンペイは、後手必勝が知られているため、既存ルールを細かく調整するのではなく、追加ルールによってゲーム性を変える。

この拡張ルールでは、以下の3つの要素を追加する。

- 障害物トークン
- 特性付きコマ
- サイズ付きコマと被せるルール

目的は以下。

- 定跡通りに進みにくくする
- コマごとの役割を増やす
- 盤面上の見えているコマと隠れているコマの読み合いを作る
- 挟んで飛ばす以外の戦略を増やす
- 毎局の展開に幅を持たせる

## 2. 拡張後のコマ構成

各プレイヤーは、通常の4個のコマの代わりに、以下の4個のコマを持つ。

| コマ | サイズ | 特性 | 個数 |
| --- | --: | -- | -: |
| 大コマ | 3 | 重い | 1個 |
| 中コマ | 2 | 標準 | 1個 |
| 小コマ | 1 | 軽い | 2個 |

つまり、各プレイヤーは以下の4個を使う。

```text
大コマ × 1
中コマ × 1
小コマ × 2
```

コマの所有者は色で区別する。
同じサイズ・同じ特性のコマは、ゲーム上同じ性能を持つ。

## 3. サイズの基本ルール

コマにはサイズがある。

| サイズ | 呼び方 |
| --: | --- |
| 3 | 大コマ |
| 2 | 中コマ |
| 1 | 小コマ |

サイズが大きいコマは、サイズが小さいコマの上に被せることができる。

```text
大コマ → 中コマ・小コマに被せられる
中コマ → 小コマに被せられる
小コマ → 他のコマに被せられない
```

同じサイズ以上のコマには被せられない。

```text
大コマ → 大コマには被せられない
中コマ → 中コマ・大コマには被せられない
小コマ → 小コマ・中コマ・大コマには被せられない
```

## 4. 被せるルール

自分のコマを、より小さいコマがある地点に移動・配置することで、そのコマの上に被せることができる。

被せる対象は、自分のコマでも相手のコマでもよい。

```text
自分の大コマを、相手の小コマの上に置く → 可能
自分の中コマを、自分の小コマの上に置く → 可能
自分の小コマを、相手の中コマの上に置く → 不可
```

## 5. スタックの考え方

コマが被さった地点は、スタックとして扱う。

```text
上
---
大コマ
小コマ
---
下
```

ゲーム上、基本的に意味を持つのは一番上に見えているコマだけである。
これを「トップコマ」と呼ぶ。

## 6. トップコマの扱い

勝利判定・挟み判定・移動対象として扱うのは、各地点のトップコマだけである。

下に隠れているコマは、以下のように扱う。

| 判定 | 隠れているコマの扱い |
| --- | --- |
| 勝利判定 | 数えない |
| 挟み判定 | 数えない |
| 移動対象 | 選択できない |
| 所有者判定 | トップコマだけを見る |

例えば、相手の小コマの上に自分の大コマを被せた場合、その地点は自分のコマがある地点として扱う。

```text
見えている状態:
自分の大コマ

下に隠れている状態:
相手の小コマ

この地点の所有者:
自分
```

## 7. 被せられたコマの復活

トップコマが移動した場合、その下にあるコマが再び見えるようになる。

```text
移動前:
大コマ
小コマ

大コマが移動した後:
小コマ
```

このとき、下から現れたコマは通常のコマとして扱う。

ただし、相手の手番中に自分のコマが現れて3個並んでも、それだけでは勝利にならない。

## 8. 配置フェーズでの被せ

配置フェーズでは、手元のコマを以下の地点に置ける。

- 空いている地点
- 自分より小さいトップコマがある地点

ただし、先手の1手目は既存ルール通り、上の世界の中央4か所の空き地点に置く。

```text
先手1手目:
U11, U12, U21, U22 の空き地点のみ
```

先手1手目では被せはできない。

## 9. 移動フェーズでの被せ

移動フェーズでは、通常の移動先として以下を選べる。

- 隣接する別世界の空き地点
- 隣接する別世界にある、自分より小さいトップコマの地点

つまり、移動先に相手のコマがあっても、自分のコマの方が大きければ、その上に被せられる。

## 10. 大コマの特性

大コマは「重いコマ」として扱う。

### 特性

大コマは、挟まれても遠くへ飛ばされにくい。

通常ルールでは、挟まれたコマは任意の空き地点へ飛ばされる。
しかし、大コマが挟まれた場合、飛ばし先は元の位置に隣接する空き地点に限定される。

```text
通常コマ:
任意の空き地点へ飛ばされる

大コマ:
元の位置に隣接する空き地点にしか飛ばされない
```

### 隣接空き地点がない場合

大コマの飛ばし先となる隣接空き地点がない場合、その大コマは飛ばされない。

この場合、挟みは成立するが、飛ばし効果は無効になる。

## 11. 中コマの特性

中コマは標準コマとして扱う。

特別な移動能力や防御能力は持たない。

ただし、サイズ2なので、小コマに被せることができる。

```text
中コマ → 小コマに被せられる
中コマ → 中コマ・大コマには被せられない
```

## 12. 小コマの特性

小コマは「軽いコマ」として扱う。

### 特性

小コマは、ゲーム中1回だけ、2ステップ移動ができる。

通常の移動では、隣接する別世界へ1ステップだけ移動する。
小コマの特殊移動では、以下のように2ステップ移動できる。

```text
上の世界 → 下の世界 → 上の世界
下の世界 → 上の世界 → 下の世界
```

### 2ステップ移動の条件

2ステップ移動では、以下を満たす必要がある。

- 1ステップ目が合法である
- 2ステップ目も合法である
- 最終地点が空き地点、または自分より小さいトップコマの地点である
- 途中地点に止まるわけではない
- 挟み判定は最終地点だけで行う

### 使用回数

小コマの2ステップ移動は、各小コマにつきゲーム中1回だけ使える。

各プレイヤーは小コマを2個持つため、最大2回まで特殊移動できる。

## 13. 障害物トークン

各プレイヤーは、障害物トークンを1個持つ。

障害物トークンは、盤面上の地点を使用不能にするためのトークンである。

## 14. 障害物トークンの配置タイミング

障害物トークンは、移動フェーズでのみ使用できる。

自分の手番で、通常の移動の代わりに、障害物トークンを空き地点に置くことができる。

```text
通常の手番:
自分のコマを移動する

障害物使用時:
自分のコマを移動せず、障害物を置く
```

障害物を置いたら、その手番は終了する。

## 15. 障害物トークンを置ける場所

障害物トークンは、以下の地点に置ける。

- 空いている地点
- コマがない地点
- まだ障害物が置かれていない地点

以下の地点には置けない。

- 自分のコマがある地点
- 相手のコマがある地点
- スタックがある地点
- すでに障害物がある地点

## 16. 障害物トークンの効果

障害物トークンがある地点は、使用不能になる。

その地点には、以下の操作ができない。

- コマを配置する
- コマを移動する
- コマを被せる
- 挟まれたコマを飛ばす
- 障害物を重ねて置く

障害物トークンはコマではないため、勝利判定や挟み判定には使わない。

## 17. 障害物トークンの移動・除去

v0.1では、障害物トークンは一度置いたら移動できない。

また、除去もできない。

```text
障害物:
置いたらゲーム終了までその地点を塞ぐ
```

## 18. 勝利判定

勝利判定は、既存のシンペイと同じく、同じ世界の中に自分のコマを3個一直線に並べることで行う。

ただし、拡張ルールでは、勝利判定に使うのはトップコマだけである。

```text
見えている自分のコマ3個が一直線
→ 勝利
```

下に隠れているコマは勝利判定に使わない。

## 19. 4個並び

4個並びは、既存ルールと同じく勝利ではない。

見えているトップコマが4個一直線に並んでいる場合、それは勝利ではない。

```text
自分のトップコマが3個ちょうど並ぶ
→ 勝利

自分のトップコマが4個並ぶ
→ 勝利ではない
```

## 20. 挟み判定

挟み判定も、トップコマだけを見て行う。

下に隠れているコマは、挟み判定の対象にならない。

```text
自分のトップコマ - 相手のトップコマ - 自分のトップコマ
```

この形になった場合、間にある相手のトップコマを挟んだものとして扱う。

## 21. 挟まれたスタックの扱い

スタックの一番上にあるコマが挟まれた場合、飛ばされるのはトップコマだけである。

下にあるコマはその場に残る。

```text
移動前:
相手の大コマ
自分の小コマ

相手の大コマが挟まれて飛ばされた後:
自分の小コマ
```

このように、トップコマが飛ばされると、下に隠れていたコマが再び見える。

下から現れたコマが3個並びを作っても、それだけでは勝利にならない。
また、下から現れたコマによって新しい挟み形ができても、連鎖挟みは発生しない。
勝利判定と挟み判定は、手番プレイヤーの主アクションで動いた先を起点に一度だけ行う。

## 22. 飛ばし先での被せ

v0.1では、挟まれて飛ばされるコマの移動先は、空き地点のみとする。

飛ばしによって、他のコマの上に被せることはできない。

```text
通常移動:
被せ可能

挟みによる飛ばし:
空き地点のみ
```

これは、飛ばし処理が強くなりすぎるのを防ぐためである。

## 23. 勝利判定と挟み処理の順序

1手番の処理順序は以下とする。

```text
1. 手番プレイヤーが主アクションを行う
   - 配置
   - 移動
   - 被せ
   - 小コマの2ステップ移動
   - 障害物トークンの配置

2. 手番プレイヤーの勝利判定を行う

3. 勝利していない場合、挟み判定を行う

4. 挟みが成立していれば、対象の相手コマを飛ばす

5. 手番を交代する
```

飛ばし処理の結果、どちらかのコマが3個並んでも、それだけでは勝利にならない。

勝利になるのは、自分の主アクションによって3個並びを作った場合のみである。
トップコマを動かした元の地点で下のコマが現れて3個並びになった場合も、勝利にはならない。

## 24. 複数のコマを同時に挟んだ場合

1手で複数の相手コマを挟んだ場合、挟まれたトップコマをすべて飛ばす。

複数の対象がある場合、手番プレイヤーが飛ばす順番を選ぶ。

ただし、大コマが含まれる場合、大コマの飛ばし先制限を適用する。

## 25. 障害物と挟みの関係

障害物はコマではないため、挟めない。

```text
自分のコマ - 障害物 - 自分のコマ
```

この形になっても、挟みは発生しない。

また、障害物を挟み判定の自分側のコマとして使うこともできない。

## 26. 障害物と勝利判定の関係

障害物は勝利判定に使わない。

障害物がライン上にあっても、3個の自分のトップコマが並んでいれば勝利する。

ただし、障害物がある地点にはコマを置けないため、その地点を使ったラインは作れない。

## 27. 使用例

### 例1: 大コマで相手の小コマに被せる

```text
移動前:
L11 = 相手の小コマ

自分の大コマを L11 に移動

移動後:
L11 = 自分の大コマ
      相手の小コマ
```

この場合、L11 は自分の地点として扱う。

### 例2: 被せていた大コマが移動する

```text
移動前:
L11 = 自分の大コマ
      相手の小コマ

自分の大コマが別の地点へ移動

移動後:
L11 = 相手の小コマ
```

隠れていた相手の小コマが再び見える。

### 例3: 小コマの2ステップ移動

```text
移動前:
U11 = 自分の小コマ

2ステップ移動:
U11 → L00 → U00

移動後:
U00 = 自分の小コマ
```

挟み判定は、最終地点である U00 だけで行う。

### 例4: 障害物を置く

```text
自分の手番で、移動の代わりに障害物を U12 に置く。

以後、U12 にはコマを置けない。
U12 へ移動できない。
U12 へ飛ばすこともできない。
```

## 28. 実装で追加すべき状態

```ts
type Player = "red" | "blue";

type PieceSize = 1 | 2 | 3;

type PieceTrait = "heavy" | "normal" | "light";

type Piece = {
  id: string;
  owner: Player;
  size: PieceSize;
  trait: PieceTrait;
  hasUsedSpecialMove?: boolean;
};

type Cell = {
  stack: Piece[];
  obstacle: boolean;
};

type Board = Record<PositionId, Cell>;
```

## 31. まず検証したいバランス項目

この拡張ルールでは、以下を検証する。

- 先手勝率
- 後手勝率
- 引き分け率
- 平均手数
- 大コマの強さ
- 小コマの2ステップ移動の強さ
- 障害物トークンの使用タイミング
- 被せルールによってゲームが複雑になりすぎないか
- 隠れたコマの復活が分かりやすいか
- 1局の時間が長くなりすぎないか

## 32. v0.1での推奨設定

最初の実装では、以下の設定にする。

```text
障害物トークン:
各プレイヤー1個
移動フェーズのみ使用可能
通常移動の代わりに置く
一度置いたら動かせない

コマ構成:
大コマ1個
中コマ1個
小コマ2個

大コマ:
挟まれても隣接空き地点にしか飛ばされない
隣接空き地点がなければ飛ばされない

中コマ:
標準性能
小コマに被せられる

小コマ:
各コマ1回だけ2ステップ移動できる

被せ:
自分より小さいトップコマに被せられる
自分のコマにも相手のコマにも被せられる
勝利判定・挟み判定はトップコマだけを見る

飛ばし:
飛ばされるのはトップコマだけ
飛ばし先は空き地点のみ
飛ばしでは被せられない
```
