# simpei rule

## 1. 概要

シンペイは、2人用の完全情報アブストラクトゲームである。

プレイヤーは赤・青に分かれ、それぞれ4個の駒を使う。
盤面は「上の世界」と「下の世界」の2層構造になっている。

## 2. 使用するもの

| 要素     | 内容            |
| ------ | ------------- |
| プレイヤー数 | 2人            |
| 駒      | 赤・青それぞれ4個     |
| 上の世界   | 4×4の交点        |
| 下の世界   | 3×3の交点        |
| 置ける場所  | 合計25か所        |
| 1マスの制約 | 1か所に置ける駒は1個まで |

## 3. 勝利条件

自分の手番で、同じ世界の中に自分の駒を3個一直線に並べると勝ち。

一直線とは、以下のいずれかである。

* 縦
* 横
* 斜め

### 勝利条件の注意点

| 状態                 | 勝ちになるか |
| ------------------ | ------ |
| 上の世界だけで3個並ぶ        | 勝ち     |
| 下の世界だけで3個並ぶ        | 勝ち     |
| 上の世界と下の世界をまたいで3個並ぶ | 勝ちではない |
| 4個並ぶ               | 勝ちではない |
| 相手の手番中に自分の3個並びができる | 勝ちではない |

勝利判定は「自分の手番で3個並べたか」を見る。
相手に駒を飛ばされた結果、自分の駒が3個並んでも勝ちにはならない。

## 4. ゲーム開始

最初は、両プレイヤーが自分の駒4個を手元に持っている。

先手・後手が交互に手番を行う。

## 5. 配置フェーズ

最初の8手は配置フェーズである。

各プレイヤーは、自分の手番で手持ちの駒を1個、空いている場所に置く。

| 手数      | 内容            |
| ------- | ------------- |
| 1手目〜8手目 | 手持ちの駒を空き場所に置く |
| 9手目以降   | 盤上の自分の駒を移動する  |

### 配置フェーズの特別ルール

先手の1手目は、上の世界の中央4か所のいずれかに置かなければならない。

それ以降は、空いている場所であれば基本的にどこに置いてもよい。

## 6. 移動フェーズ

両プレイヤーが4個ずつ、合計8個の駒を置き終わっても決着しない場合、9手目以降は移動フェーズになる。

移動フェーズでは、自分の手番に以下を行う。

1. 盤上にある自分の駒を1個選ぶ
2. 隣接する別の世界の空き場所へ移動する

## 7. 移動ルール

移動は、必ず「別の世界」へ行う。

| 元の場所   | 移動先      |
| ------ | -------- |
| 上の世界の駒 | 隣接する下の世界 |
| 下の世界の駒 | 隣接する上の世界 |

### 移動ルールの注意点

* 上の世界から上の世界へは移動できない
* 下の世界から下の世界へは移動できない
* 移動先は空いている必要がある
* 移動できるのは、盤面上で隣接している場所だけである

## 8. 移動できない場合

自分の手番で、どの自分の駒も合法的に移動できない場合はパスになる。

パスしたら、何もせず相手の手番になる。

## 9. 挟むルール

自分が駒を置いた、または移動した結果、同じ世界にある相手の駒を自分の駒で挟んだ場合、その相手の駒を別の空き場所へ移動させる。

イメージは以下。

```text
自分の駒 - 相手の駒 - 自分の駒
```

このような形になった場合、間にある相手の駒を飛ばす。

## 10. 挟み判定

挟み判定は、同じ世界の中だけで行う。

| 状態                    | 挟めるか |
| --------------------- | ---- |
| 上の世界の自分駒で、上の世界の相手駒を挟む | 挟める  |
| 下の世界の自分駒で、下の世界の相手駒を挟む | 挟める  |
| 上の世界と下の世界をまたいで挟む      | 挟めない |

挟み方向は、勝利条件と同じく以下のいずれかである。

* 縦
* 横
* 斜め

## 11. 挟んだ駒の移動先

挟まれた相手の駒は、任意の空き場所へ移動させる。

通常の移動とは異なり、挟まれた駒を飛ばすときは以下の制約を受けない。

* 隣接している必要はない
* 上の世界から下の世界、下の世界から上の世界、という制約はない
* 同じ世界に飛ばしてもよい
* 別の世界に飛ばしてもよい

### 挟んだ駒を飛ばすときの制約

| 制約         | 内容                          |
| ---------- | --------------------------- |
| 元の場所には戻せない | 必ず元の場所とは別の空き場所に飛ばす          |
| 空き場所に置く    | すでに駒がある場所には置けない             |
| 相手の3個並び    | 飛ばした結果、相手が3個並んでも相手の勝ちではない   |
| 連鎖         | 飛ばした結果、自分の駒が挟まれる形になっても連鎖しない |

## 12. 複数の駒を挟んだ場合

1手で相手の駒を複数挟んだ場合、挟まれた駒はすべて飛ばす。

例えば、1回の手番で2個以上の相手駒を同時に挟んだ場合、それらをすべて別の空き場所に移動させる。

それぞれの駒は、飛ばす前の位置とは別の空き場所に置く必要がある。

## 13. 4個並びについて

シンペイでは、3個ちょうどの一列が勝利条件である。

そのため、4個並んでいても勝ちではない。

また、4個並んでいる状態から1個を動かして、結果的に3個並びが残ったとしても、それは勝ちにならないとされている。

## 14. 手番の流れ

### 配置フェーズの手番

1. 手番プレイヤーが手持ちの駒を1個選ぶ
2. 空いている場所に置く
3. 勝利条件を満たしたか確認する
4. 相手の駒を挟んだか確認する
5. 挟んでいた場合、挟まれた相手の駒を空き場所へ飛ばす
6. 手番を交代する

### 移動フェーズの手番

1. 手番プレイヤーが盤上の自分の駒を1個選ぶ
2. 隣接する別の世界の空き場所へ移動する
3. 勝利条件を満たしたか確認する
4. 相手の駒を挟んだか確認する
5. 挟んでいた場合、挟まれた相手の駒を空き場所へ飛ばす
6. 手番を交代する

### 移動できない場合

1. 手番プレイヤーが合法手を持たない
2. パスする
3. 手番を交代する

## 15. 実装時に持つべき状態

実装する場合、最低限以下の状態を持つとよい。

| 状態                  | 内容             |
| ------------------- | -------------- |
| `board`             | 25か所それぞれの占有状態  |
| `currentPlayer`     | 現在の手番プレイヤー     |
| `turnNumber`        | 現在の手数          |
| `placedCount`       | 各プレイヤーが何個置いたか  |
| `phase`             | 配置フェーズか移動フェーズか |
| `pendingForcedMove` | 挟まれた駒を飛ばす処理中か  |
| `winner`            | 勝者がいるか         |

## 16. 実装用ルール要約

```text
シンペイは2人用の完全情報ゲームである。

プレイヤーは赤と青に分かれ、それぞれ4個の駒を持つ。
盤面は上の世界4×4と下の世界3×3からなり、合計25個の交点がある。
各交点には駒を1個だけ置ける。

勝利条件は、自分の手番で、同じ世界の中に自分の駒を縦・横・斜めのいずれかに3個一直線に並べることである。
上の世界と下の世界をまたいだ3個並びは勝ちではない。
4個並びは勝ちではない。

最初の8手は配置フェーズである。
各プレイヤーは手持ちの駒を交互に空き交点へ置く。
ただし、先手の1手目は上の世界の中央4交点のいずれかに置かなければならない。
それ以降は空き交点ならどこに置いてもよい。

9手目以降は移動フェーズである。
手番プレイヤーは、自分の駒を1個選び、隣接する別の世界の空き交点へ1ステップ移動する。
上の世界の駒は隣接する下の世界へ、下の世界の駒は隣接する上の世界へ移動する。
同じ世界内には移動できない。
移動可能な駒がない場合はパスする。

自分が駒を置く、または移動した結果、同じ世界にある相手の駒を自分の駒で縦・横・斜めに挟んだ場合、挟まれた相手の駒を任意の空き交点へ移動させる。
挟み判定では上の世界と下の世界を独立して扱う。
挟まれた駒の移動先は元の場所とは別でなければならない。
複数の相手駒を同時に挟んだ場合、それらをすべて移動させる。
挟んだ駒を移動させた結果、相手の3個並びができても相手の勝ちにはならない。
挟まれた駒を移動させた結果、自分の駒が挟まれる形になっても連鎖処理は発生しない。
```

## 17. 要確認事項

実装前に確認した方がよい点は以下。

* 同一局面の繰り返しをどう扱うか

  * 公式ルール上、明確な引き分け規定があるか要確認
  * 実装上は「同一局面が一定回数出たら引き分け」などを独自ルールとして入れる可能性がある

* 複数の駒を挟んだ場合の飛ばす順序

  * 実装では、挟まれた駒の一覧を作り、順番に移動先を選ばせると扱いやすい

* 挟んだ直後の勝利判定順序

  * 基本的には、自分の着手によって3個並びができたかを確認する
  * その後に挟み処理を行う形にすると実装しやすい
